Analyse

Notre but est que la plupart des cas d’utilisation seront informatisés dans le système (ex: Passer une commande, mettre un produit en vente) via la programmation orientée objet avec Java pour représenter: Client, Commande, Produit, etc. Les interventions extérieures au système comme le service de livraison ne seront pas informatisées. Les données seront stockées dans un serveur en Java contenant une liste pour les identifiants des clients ainsi que les inventaires de tous les revendeurs. Comme le produit final sera un fichier.jar, le matériel nécessaire sera un ordinateur personnel équipé de Java, d’un disque dur interne, et d’une interface en ligne de commande. Il faudra également un disque dur externe, pour stocker les données. Comme le serveur devrait être en ligne pour permettre la connexion de plusieurs usagers et pour préserver les données, UniShop interagira avec les autres systèmes via Internet et le cloud pour la sécurité. Les utilisateurs interagiront avec le système via l’interface en lignes de commande de leur PC (ils devront être connectés à UniShop pour pouvoir utiliser la plateforme complète).
