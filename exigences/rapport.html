Cas d'utilisation

  1. Cas d'utilisation : S’inscrire à la plateforme en tant qu’acheteur.
    But : Un acheteur souhaite s’inscrire à la plateforme nécessaires.
    Préconditions :L’acheteur dispose d’un nom, prénom, pseudo, adresse courriel, téléphone et un adresse d’expédition.
    Acteurs : Acheteur (Principal) (secondaire).
    Scénario principal :
    1. Acheteur se rend à la plateforme.
    2. Acheteur pèse sur Inscription
    3. Acheteur entre ses informations personnelles (nom, prénom, pseudo(unique), adresse courriel, téléphone et adresse d’expédition) dans le système.
    4. Le système vérifier la validité du courriel et du pseudo unique.
    5. Le système affiche Succès à l’écran.
    6. Le système demande de cocher la boîte pour accepter les termes et conditions.
    7. Acheteur coche la boîte
    8. Acheteur pèse le bouton S’inscrire pour poursuivre à la création.
    9. Le système confirme la création du compte.
    Scénarios alternatifs :
    4a. Le courriel n’est pas valide (mal saisi le courriel).
    4a.1 Le système redemande de nouveau l’entrée d’adresse courriel.
    4a.2 Acheteur saisit de nouveau son adresse courriel.
    4a.3 La scène recommence à l’étape 4.
    4b. Le pseudo n'est pas unique.
    4b.1 Le système redemande à l’acheteur de saisir un pseudo qui est unique de nouveau.
    4b.2 L’acheteur saisit de nouveau un pseudo.
    4b.3 La scène recommence à l’étape 4.
    6a. L’acheteur n’accepte pas les termes et conditions.
    6a.1 L’acheteur pèse le bouton S’inscrire pour poursuivre à la création.
    6a.2 Le système envoie Erreur: Ce champ est obligatoire.
    6a.3 Retour à l’étape 6.
    Postconditions : L’acheteur a créé un compte et peut modifier son profil après l’inscription.

  2. Cas d'utilisation : S’inscrire à la plateforme en tant que revendeur.
    But : Un revendeur souhaite s’inscrire à la plateforme nécessaires.
    Préconditions :Au moins un produit à revendre.
    Acteurs : Revendeur (Principal)
    Scénario principal :
    1. Revendeur se rend à la plateforme.
    2. Revendeur pèse sur Inscription.
    3. Revendeur entre ses informations personnelles (nom unique, adresse, adresse courriel, téléphone) dans le système.
    4. Le système vérifier la validité du courriel et du nom unique.
    5. Le système affiche Succès à l’écran.
    6. Le système demande de cocher la boîte pour accepter les termes et conditions.
    7. Revendeur coche la boîte
    8. Revendeur pèse le bouton S’inscrire pour poursuivre à la création.
    9. Le système confirme la création du compte.
    Scénarios alternatifs :
    4a. Le courriel n’est pas valide (mal saisi le courriel).
    4a.1 Le système redemande de nouveau l’entrée d’adresse courriel.
    4a.2 Revendeur saisit de nouveau son adresse courriel.
    4a.3 La scène recommence à l’étape 4.
    4b. Le nom n'est pas unique.
    4b.1 Le système redemande au revendeur de saisir un nom qui est unique de nouveau.
    4b.2 Le revendeur saisit de nouveau un nom.
    4b.3 La scène recommence à l’étape 4.
    Postconditions : Le revendeur doit modifier son profil après l’inscription.

  3. Cas d'utilisation : Se connecter à la plateforme.
    But : Acteur désire accéder au compte sur la plateforme.
    Préconditions :Au moins un produit à revendre.
    Acteurs :Acheteur (principal), Revendeur (Principal)
    Scénario principal :Le client doit disposer d’un compte.
    1. Le client se présente à la page d'accueil.
    2. Le client pèse le bouton Connexion.
    3. Le système lui demande pour son nom d’utilisateur et son courriel.
    4. Le client entre son nom d’utilisateur et son courriel.
    5. Le système vérifie que les données correspondent à ceux dans le système.
    6. Le système sauvegarde les modifications.
    7. Le système affiche Succès à l’écran.
    8. Le système laisse entrer le client à la plateforme.
    Scénarios alternatifs :
    2a. Le client veut se connecter en tant qu’acheteur.
    2a.1 Le client pèse sur Connexion en-dessous de Acheteur.
    2a.2 La scène reprend à l’étape 3.
    2b. Le client veut se connecter en tant que revendeur.
    2b.1 Le client pèse sur Connexion en-dessous de Revendeur.
    2b.2. La scène reprend à l’étape 3.
    5a. Le nom d’utilisateur n’est pas valide (nom a été mal saisi).
    5a.1 Le système n’a pas trouvé le compte.
    5a.2 Le système affiche Compte inexistant à l’écran.
    5a.3 Le système reprend à l’étape 4.
    5b. Le courriel n’est pas valide (courriel a été mal saisi).
    5b.1 Le système n’a pas trouvé le compte.
    5b.2 Le système affiche Compte inexistant à l’écran.
    5b.3 Le système reprend à l’étape 4.
    Postconditions : Le système enregistre amène l’acheteur/ le revendeur à la plateforme.

  4. Cas d'utilisation : Modifier son profil.
    But : Modifier le profil.
    Préconditions :Au moins un produit à revendre.
    Acteurs :Acheteur (principal), Revendeur (Principal)
    Scénario principal :Le client est inscrit sur la plateforme.
    1. Le client sélectionne l’option “modifier le profil” dans le réglage.
    2. Le client choisit et modifie le champ à modifier dans leur profil.
    3. Le système vérifie les informations modifiées.
    4. Le système sauvegarde les informations modifiées.
    5. Le système affiche Succès à l’écran.
    6. Le système envoie une notification du changement.
    7. Le client pourra quitter ou continuer sur la plateforme.
    Scénarios alternatifs :
    3a. Le client laisse un champ vide.
    3a.1 Le système affiche Erreur: Le champ est obligatoire.
    3a.2 Le client va remplir le champ manqué.
    3a.3 La scène reprend à l’étape 3.
    Postconditions : Être capable de modifier le profil encore.

  5. Cas d'utilisation : Rechercher un/des produit(s) ou un/des revendeurs.
    But : Recherche par mot-clé pour trouver des produits spécifiques ou un revendeur.
    Préconditions :- Accès à la barre de recherche;
    - L’acheteur est inscrit à la plateforme;
    - L’acheteur est connecté à la plateforme.
    Acteurs :Acheteur (Principal)
    Scénario principal :Le client est inscrit sur la plateforme.
    1. L’acheteur se présente à la page d’accueil.
    2. L’acheteur sélectionne quel type de recherche il veut faire.
    3. L’acheteur pèse le champ de recherche.
    4. L’acheteur écrit le(s) mot-clé(s) dans la barre de recherche.
    5. L’acheteur pèse sur Rechercher.
    6. Le système fait la recherche par catégorie, prix, note moyenne ou popularité.
    7.Le système amène le client à une page avec une liste de résultats possibles.
    Scénarios alternatifs :
    2a. L’acheteur veut rechercher un produit.
    2a.1 L’acheteur sélectionne Produit.
    2a.2 La scène reprend à l’étape 4.
    2b. L’acheteur veut rechercher un revendeur.
    2b.1 L’acheteur sélectionne Revendeur.
    2b.1 La scène reprend à l’étape 4.
    6a. Le(s) mot(s) clé(s) invalide(s) (mal saisi le mot).
    6a.1 Le système trouve aucun résultat correspondant aux mots-clés.
    6a.2 Le système affiche Aucun résultat sur l’écran.
    6a.3 La scène reprend à l’étape 4.
    7a. Le système affiche une liste de produits possibles.
    7a.1 L’acheteur utilise le système de filtrage pour faciliter sa recherche.
    7a.2 L'acheteur trouve le produit recherché..
    7a.3 Le scénario se termine.
    7b. Le système affiche une liste de revendeurs possibles.
    7b.1 L’acheteur trouve le revendeur recherché..
    7b.2 Le scénario se termine
